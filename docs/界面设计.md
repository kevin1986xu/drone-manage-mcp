**日期**：2026-07-10

## 一、产品形态

单页双栏（Claude Artifacts / Codex Cowork 式 "Chat + Canvas"）：

```
┌────────────────────┬──────────────────────────────────┐
│  左栏：对话框         │  右栏：效果区（浏览器视图）           │
│                    │                                  │
│  · 自然语言输入      │  随 Agent 工具调用自动切换：          │
│  · Agent CoT 步骤   │  · 地图（图斑/无人机/航线）           │
│    流式渲染          │  · 免登录嵌入的航线编辑界面           │
│  · 工具调用状态      │  · 飞前检查报告卡片                  │
│  · 放飞确认卡片      │  · （P1）无人机直播流/遥测面板        │
└────────────────────┴──────────────────────────────────┘
```

MCP/CLI 完全藏在 Agent 背后，用户全程只面对这个分屏，**不切页面、不点菜单**。

## 二、协议选型：AG-UI（Agent-User Interaction Protocol）

- 开源、事件驱动、SSE 传输的 agent↔前端标准协议（CopilotKit 主导），**已被 Google、LangChain、AWS、Microsoft、PydanticAI 采用**。
- **LangGraph 每个节点可原生 emit AG-UI 事件**——与生产期编排选型无缝衔接。
- 三协议分工（向客户讲架构的清晰图景）：**MCP = Agent↔工具，A2A = Agent↔Agent，AG-UI = Agent↔用户界面**。
- 事件类型覆盖全部需要：消息流式输出、工具调用开始/结束（→ 左栏 CoT 步骤）、状态同步（→ 右栏视图数据）、human-in-the-loop 请求（→ 确认卡片）。

## 三、前端技术栈（全部开源）

| 层 | 选型 | 协议 | 说明 |
|---|---|---|---|
| 基础框架 | React 18 + TypeScript + Vite | MIT | CopilotKit 及主流 agent 前端生态均 React 优先 |
| Agent 对话层 | **CopilotKit** | MIT | AG-UI 官方实现；备选 assistant-ui（更轻，只要对话组件时用） |
| 地图（演示期） | **MapLibre GL JS（2D）** | BSD | 图斑/机标/航线 2D 足够表达；底图接天地图 WMTS，演示区瓦片本地缓存防断网 |
| 地图（生产期） | **Cesium（3D）** | Apache 2.0 | 任务监控场景必需：航高/仿地/净空只有 3D 讲得清；与 MapLibre 双引擎共存 |
| 地图（客户有 SuperMap 底座） | SuperMap iClient3D | 商业 | 基于 Cesium 魔改，迁移成本可控；进场前先确认客户底座 |
| UI 组件库 | Ant Design | MIT | 政企事实标准，暗色主题齐全 |
| 状态管理 | Zustand | MIT | 管"右栏视图指令 + 地图图层状态" |
| 事件流 | @microsoft/fetch-event-source | MIT | 支持 POST、断线重连、自定义鉴权 header |

**关键架构决策（D1 即定）**：地图操作封装成自有接口层（`addPlots(geojson)` / `drawRoute(route)` / `flyTo(target)`），底下 MapLibre/Cesium/iClient3D 对上层不可见——三个场景通吃的保险。

**后端衔接说明**：AG-UI 官方集成不含 Qwen-Agent；国产路线后端用 **LangGraph 最小图 + Qwen 模型**（AG-UI 一等公民，零胶水），详见《技术实现路线推荐》§3.1。

**交互原型**：`界面原型.html`（本目录，纯 HTML 零依赖，仅定交互与视觉，非生产代码），在线版 https://claude.ai/code/artifact/7ee36fc5-03dc-4717-89ad-c8f853299b2a ——完整可点的 7.22 演示动线，含 AG-UI 事件标注。

## 四、左栏：CopilotKit（React）

- AG-UI 官方前端实现：现成对话组件 + 流式渲染 + **内置 human-in-the-loop 确认组件**（放飞审批卡片不用自己写）+ generative UI（工具结果渲染为卡片而非纯文本）。
- 12 天冲刺下比自研快。若坚持自研：本质是"一个 SSE 端点 + 5 种自定义事件的 React 双栏页"，工作量可控，但 HITL 组件要自己做。
- 前端人力收敛：右栏复用现有页面（iframe），前端新增工作量集中在对话框一个组件上——对冲录音识别的前端人力风险。

## 五、右栏：视图指令事件驱动

Agent 工具调用结果通过 AG-UI 的 state/custom 事件映射为右栏视图切换：

| 视图指令 | 触发工具 | 右栏内容 |
|---|---|---|
| `show_map(geojson)` | query_plots / find_nearby_drones / generate_route | 地图组件渲染图斑、无人机位置、航线（政企 GIS 栈优先 SuperMap，或 Cesium） |
| `show_iframe(url)` | open_route_editor | 免登录 token URL 嵌入现有航线编辑界面 |
| `show_report(data)` | preflight_check | 飞前检查逐项打勾卡片（天气/电量/避障/空域） |
| `show_confirm(action)` | dispatch_drone / take_off | 放飞确认卡片，人点击后 Agent 继续（LangGraph interrupt 恢复） |
| `show_stream(url)` | （P1）任务监控 | 直播流 / 遥测面板 |

**编辑闭环**：iframe 内编辑完成 → 编辑页 postMessage 回传 → 前端转为 AG-UI 事件送回 Agent → Agent 继续后续步骤。

**备注**：MCP 2026-07 新版的 MCP Apps 扩展（工具直接返回服务端渲染 UI）方向一致但生态尚早——暂不采用，保持关注。

## 六、7.22 演示动线（现场即此界面）

1. 左栏输入"帮我查一下XX区域的图斑" → 左栏流式："正在查询图斑…找到 5 个" + 右栏地图落点
2. "调度周边无人机" → 右栏地图出现 3 架无人机图标
3. "规划航线" → 右栏画出航线 + 左栏 Agent 解释覆盖逻辑（"周边还有 2 个图斑，本航线刚好可覆盖"）
4. "我调整一下" → 右栏切入航线编辑界面 → 编辑完成右栏回到地图
5. "我要起飞" → 左栏逐项 CoT 检查（天气/电量/避障/空域）+ 右栏检查报告
6. 确认卡片 → 人点击放飞 → 演示结束

**全程一个界面、一条对话流。"AI 自主规划 + 人掌握最终决定权"本身就是演示加分项。**
