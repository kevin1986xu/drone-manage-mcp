# 正式版系统启动手册

> 面向本机测试。分两档：**A 核心对话链路**（最小可用，跑通"对话→工具→平台"）、
> **B 完整链路**（+ 通用前端页面 + 对外网关）。先按 A 跑通，再按需加 B。
> 命令都在 `正式开发/` 目录下执行（下称项目根）。

## 0. 服务清单

| # | 服务 | 端口 | venv / 运行时 | 档 | 说明 |
|---|---|---|---|---|---|
| 1 | mcp-services（十一域 MCP） | 8201-8204、8206-8212 | `mcp-services/.venv` (py3.13) | A | 业务工具，真实平台直连 |
| 2 | 审批服务 approval | 8205 | `deerflow/backend/.venv` (py3.12) | A | confirm_token 唯一签发方 |
| 3 | DeerFlow Gateway | 8001 | `deerflow/backend/.venv` | A | 对话入口（LangGraph）|
| 4 | UI 服务 ui_service | 8213 | `deerflow/backend/.venv` | B | 确认卡片页/视图页（docs/08）|
| 5 | GIS BFF | 8300 | `deerflow/backend/.venv` | B | 演示版前端零改动接 DeerFlow |
| 6 | 前端（演示版） | 5173 | `../frontend`（npm） | B | 浏览器 UI |
| 7 | Higress 对外网关 | 8080 / 8888 | docker | B | 多租户鉴权/限流/审计（docs/07）|

> 两个 venv：**mcp-services 用自己的 `.venv`**，其余 Python 服务（审批/UI/BFF/网关）
> 统一用 **`deerflow/backend/.venv`**。别混。

## 1. 前置条件（缺一不可）

1. **`mcp-services/.env` 已配**（凭据不入库，本机已存在）。关键项：
   - `MCP_SERVICE_IP`：**必须是当前 en0 IP**（`ipconfig getifaddr en0` 查；漂移会导致
     Nacos 注册失效 → 网关 503，见 §6 坑①）；
   - `UAV_MCP_API_KEY`：服务间 key；`UAV_TENANT_KEYS`：多租户 key 表；
   - `DRONE_GATEWAY_BASE`+`DRONE_LOGIN_*`：回源走平台外网网关（不依赖 VPN）；
   - `NACOS_SERVER_ADDR`+账号：注册到现网 Nacos；
   - `UAV_UI_BASE=http://127.0.0.1:8213`：配了才在工具返回里带 view_url（B 档）。
2. **LLM key**：Gateway 要 `DASHSCOPE_API_KEY`，取演示版 `backend/.env` 的 `LLM_API_KEY`。
3. **网络**：本机能到现网 Nacos（`192.168.101.21:8998`）与平台外网网关
   （`demo-lt.geostar.com.cn:11412`）。先 `curl` 验通再启动。
4. **依赖已装**：两个 venv 已 `uv sync`/`pip install`；前端已 `npm install`。

## A. 核心对话链路（最小可用）

### A1. 起 mcp-services（十一域）

```bash
cd mcp-services
pkill -f uav_mcp.runner            # 清旧实例（重要，见坑②）
sleep 2
nohup .venv/bin/python -m uav_mcp.runner > /tmp/uav_mcp_runner.log 2>&1 &
# 等注册：日志出现 11 条 "Registry 已注册/已更新" 才算好
sleep 18 && grep -cE "Registry 已注册|Registry 已更新" /tmp/uav_mcp_runner.log   # 应为 11
```

### A2. 起审批服务（8205）

```bash
cd uav_extensions
# 本机测试 admin key 可留空；四眼动作可选（docs/09），如需演示删围栏四眼：
#   加 APPROVAL_FOUR_EYES_ACTIONS="delete_zone,device_reboot"
APPROVAL_ADMIN_KEY=adm-local-test \
  nohup ../deerflow/backend/.venv/bin/python -m uav_extensions.approval_service \
  > /tmp/uav_approval.log 2>&1 &
sleep 3 && curl -s localhost:8205/healthz     # {"status":"ok"}
```

### A3. 起 DeerFlow Gateway（8001）

```bash
cd deerflow/backend
DEER_FLOW_AUTH_DISABLED=1 \
  DASHSCOPE_API_KEY="<演示版 backend/.env 里的 LLM_API_KEY>" \
  UAV_MCP_API_KEY="<同 mcp-services/.env>" \
  PYTHONPATH=. \
  nohup .venv/bin/python -m uvicorn app.gateway.app:app --port 8001 \
  > /tmp/deerflow_gateway.log 2>&1 &
# 等启动 + 首次对话触发工具发现；日志应见 "loaded 74 tool(s)" 且无 "Skipping MCP server"
sleep 10 && curl -s -X POST localhost:8001/api/threads -H 'Content-Type: application/json' -d '{}'
```

### A4. 验证核心链路

```bash
# 跑一条评测（走 Gateway→MCP→平台全链路）
UAV_MCP_API_KEY="<key>" mcp-services/.venv/bin/python eval/run_eval.py 1
# 期望：✅ #1 ... query_plots  命中 1/1
```

到这里 **A 档完成**——已可用 API 与系统对话。想要浏览器 UI 或对外网关，继续 B。

## B. 完整链路（+ 通用前端 + 对外网关）

### B1. UI 服务（8213，确认卡片页/视图页）

```bash
cd uav_extensions
UAV_MCP_API_KEY="<key>" APPROVAL_BASE=http://127.0.0.1:8205 \
  nohup ../deerflow/backend/.venv/bin/python -m uav_extensions.ui_service \
  > /tmp/uav_ui.log 2>&1 &
sleep 3 && curl -s localhost:8213/healthz
```
高危工具返回里会带 `view_url`，浏览器打开即确认卡片；点确认得 `[SYSTEM_CONFIRMATION]`
一行贴回对话。（需 mcp-services/.env 配了 `UAV_UI_BASE`。）

### B2. GIS BFF（8300）+ 前端（5173）

```bash
# BFF
cd uav_extensions
UAV_MCP_API_KEY="<key>" APPROVAL_ADMIN_KEY=adm-local-test \
  nohup ../deerflow/backend/.venv/bin/python -m uav_extensions.bff \
  > /tmp/uav_bff.log 2>&1 &
sleep 3 && curl -s localhost:8300/healthz

# 前端（演示版，切后端到 BFF）
cd ../../frontend           # 注意：前端在仓库根，不在正式开发/ 下
BACKEND_PORT=8300 npm run dev      # 打开 http://localhost:5173
```

### B3. Higress 对外网关（8080 网关 / 8888 控制台）

```bash
cd deploy
docker compose -f higress-standalone.docker-compose.yml up -d
# 首次需在控制台 http://localhost:8888 配服务来源+消费者鉴权，详见 README-higress.md
# 验证（带租户 key 经网关调工具）：
curl -s -o /dev/null -w '%{http_code}\n' -X POST \
  http://localhost:8080/mcp/uav-alert-mcp/mcp \
  -H 'Content-Type: application/json' -H 'Accept: application/json, text/event-stream' \
  -H 'X-API-Key: demo-key-2026-a1b2c3' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'      # 期望 200
```
完整网关冒烟：`mcp-services/.venv/bin/python mcp-services/scripts/smoke_gateway.py`（11 域）。

## 3. 停止

```bash
pkill -f uav_mcp.runner            # mcp-services
pkill -f approval_service          # 审批
pkill -f ui_service                # UI
pkill -f uav_extensions.bff        # BFF

```

## 4. 启动顺序小结

```
A 档：mcp-services(8201..) → 审批(8205) → Gateway(8001) → 评测验证
B 档：+ UI(8213) + BFF(8300) + 前端(5173) + Higress(8080/8888)
```
依赖方向：Gateway/BFF 依赖 mcp-services 已注册；UI 依赖审批服务；网关依赖
mcp-services 注册到 Nacos。**先起被依赖方，再起依赖方。**

## 5. 关键环境变量速查

| 变量 | 用在哪 | 值 |
|---|---|---|
| `MCP_SERVICE_IP` | mcp-services/.env | 当前 en0 IP |
| `UAV_MCP_API_KEY` | 全部 Python 服务 | 服务间 key（.env 一致） |
| `UAV_TENANT_KEYS` | mcp-services/.env | 租户 key→身份表 |
| `UAV_UI_BASE` | mcp-services/.env | `http://127.0.0.1:8213`（B 档）|
| `DASHSCOPE_API_KEY` | Gateway | 演示版 `backend/.env` 的 LLM_API_KEY |
| `APPROVAL_ADMIN_KEY` | 审批/BFF | 本机可用测试值；生产必配强 key |
| `APPROVAL_FOUR_EYES_ACTIONS` | 审批 | 逗号分隔的四眼动作（docs/09，可选）|

## 6. 常见坑（都踩过）

1. **en0 IP 漂移 → 网关 503/对话命中 0**：改 `mcp-services/.env` 的 `MCP_SERVICE_IP`
   为当前 en0 → 重启 mcp-services 重注册。若 Nacos 里残留旧 IP 持久实例
   （`ephemeral:false`），删旧实例（命令见 `README-higress.md` 排障表）。
2. **runner 起不来 `[Errno 48] 8201`**：pkill 后老进程没退干净就起了新的。等 1-2 秒
   确认 `lsof -iTCP:8201 -sTCP:LISTEN` 空了再起。**别用 `lsof -ti:8201` 杀**（会误杀
   连着端口的客户端）。
3. **Gateway 调不到任何 MCP 工具、去摸 read_file**：工具发现失败被缓存。检查
   `deerflow/extensions_config.json` 的 url 是否 `127.0.0.1:820x`，然后**重启 Gateway**。
4. **改了 SKILL.md / 工具描述**：mcp-services 和 Gateway 都要重启（各有缓存）。
5. **VPN 到 192.168.101.x 抖断**：排障先 `curl` Nacos；评测单条数百秒基本是链路问题。
6. **本机部署别跑 nacos_bridge**：它会把 config 的 url 改回失效 IP 甚至清空
   mcpServers（用 127.0.0.1 版 extensions_config 免疫 IP 漂移，不需要桥）。

> 更细的：对外网关见 `README-higress.md`；架构/鉴权见 `docs/07`；通用前端 `docs/08`；
> 用户身份 `docs/09`。
